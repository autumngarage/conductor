# Cortex install baseline on conductor

**Date:** 2026-05-05
**Type:** decision
**Trigger:** human-authored

> Conductor's existing Cortex scaffold is now upgraded to the current v0.5.0 file contract, with a clean audit-instructions baseline and the remaining doctor warnings made visible.

## Context

The Cortex v0.9.0 dogfood gate requires installing Cortex on conductor with its existing `CLAUDE.md`, `AGENTS.md`, `README.md`, and `principles/` absorbed or referenced, plus a first-pass `cortex doctor --audit-instructions` baseline.

Conductor already had a `.cortex/` scaffold, but it was v0.2.3/v0.3.1-era: `SPEC_VERSION` was `0.3.1-dev`, Protocol was `0.2.0`, the release and promotion journal templates were missing, and `state.md` was still the scaffold placeholder. `cortex doctor` initially reported 16 errors, all from historical plan files that predated the current plan validator. `cortex doctor --audit-instructions` also caught stale release claims in README/agent instructions: v0.7.4/v0.3.3/v0.8.3 were still documented while GitHub releases and Homebrew both show v0.8.7.

## What we decided

Upgrade the existing scaffold in place instead of treating conductor as a fresh install. The durable conductor memory now has:

- SPEC v0.5.0 scaffold files generated by Cortex 0.8.2.
- `.cortex/config.toml` with audit-instructions checks for the Homebrew tap, local Autumn Garage siblings, conductor GitHub releases, and the repo instruction files.
- Regenerated `.cortex/state.md` from the current plan/journal/doctrine corpus.
- Historical pre-v0.5 plan files preserved under `.cortex/archive/plans/` so the single active plan surface is `plans/conductor-blindspots`.
- README and agent-instruction release claims updated to v0.8.7.
