> Conductor now owns review-gate budgets for code-review routes and handles
> OpenRouter empty responses as provider failures instead of passing blank
> output to consumers.

## What shipped

- `--auto` routes tagged `code-review` derive bounded review-gate budgets from
  prompt size and fallback count, so consumers do not have to guess raw timeout
  values for normal review delegation.
- OpenRouter empty responses retry against the remaining model stack and share
  the same remaining timeout budget before surfacing a provider error.
- Empty provider output is classified as `empty-response`, making fallback and
  supervising-agent diagnostics more explicit.
- Normal Conductor commands no longer rewrite tracked repo-scope integration
  files by default; explicit `conductor update` / `update-all` remain the
  operator path.
