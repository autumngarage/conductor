## What shipped

- Review-tagged `--auto` routes always use Conductor-derived provider timeout
  and stall budgets from prompt size and fallback breadth.
- Caller-supplied `--timeout` / `--max-stall-seconds` values are ignored for
  review-gate provider budgets and surfaced in the route diagnostic when they
  differ from the Conductor-owned values.
- CLI help and README language now steer review integrations away from customer
  timeout guessing.
- Regression coverage asserts both `conductor exec --auto --tags code-review`
  and native `conductor review --auto` use the Conductor-owned budget.
